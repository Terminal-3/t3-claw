Complete reference for the `bastionclaw` CLI.

## Global Options

```bash
bastionclaw [OPTIONS] [COMMAND]
```

| Option | Description |
|--------|-------------|
| `-c, --config <FILE>` | Path to config file |
| `--no-onboard` | Skip auto-onboarding |
| `-v, --verbose` | Enable verbose logging |
| `-h, --help` | Print help |
| `-V, --version` | Print version |

## Commands

### run

Start the BastionClaw agent.

```bash
bastionclaw run [OPTIONS]
```

Starts the agent with all configured channels. This is the main command for normal operation.

**Options:**

| Option | Description |
|--------|-------------|
| `--no-tui` | Disable TUI, use HTTP only |

**Examples:**

```bash
# Start normally
bastionclaw run

# Start without TUI
bastionclaw run --no-tui

# With verbose logging
RUST_LOG=bastionclaw=debug bastionclaw run
```

### onboard

Run the interactive setup wizard.

```bash
bastionclaw onboard [OPTIONS]
```

Configures database, LLM, channels, and security settings.

**Options:**

| Option | Description |
|--------|-------------|
| `--skip-auth` | Skip authentication steps |
| `--channels-only` | Configure only channels |

**Examples:**

```bash
# Full wizard
bastionclaw onboard

# Skip auth (use existing)
bastionclaw onboard --skip-auth

# Add new channel
bastionclaw onboard --channels-only
```

### config

Manage configuration settings.

```bash
bastionclaw config <SUBCOMMAND>
```

**Subcommands:**

| Subcommand | Description |
|------------|-------------|
| `list` | List all settings |
| `get <KEY>` | Get a specific setting |
| `set <KEY> <VALUE>` | Set a setting |
| `delete <KEY>` | Delete a setting |

**Examples:**

```bash
# List all settings
bastionclaw config list

# Get LLM backend
bastionclaw config get llm.backend

# Set model
bastionclaw config set llm.model claude-sonnet-4

# Delete setting (reset to default)
bastionclaw config delete llm.model
```

### tool

Manage tools.

```bash
bastionclaw tool <SUBCOMMAND>
```

**Subcommands:**

| Subcommand | Description |
|------------|-------------|
| `list` | List installed tools |
| `install <PATH>` | Install a tool |
| `remove <NAME>` | Remove a tool |
| `run <NAME>` | Run a tool |

**Examples:**

```bash
# List tools
bastionclaw tool list

# Install from file
bastionclaw tool install ./my-tool.wasm

# Remove tool
bastionclaw tool remove my-tool
```

### registry

Manage the tool registry.

```bash
bastionclaw registry <SUBCOMMAND>
```

**Subcommands:**

| Subcommand | Description |
|------------|-------------|
| `list` | List available tools |
| `search <QUERY>` | Search for tools |
| `install <NAME>` | Install from registry |
| `update` | Update registry index |

**Examples:**

```bash
# List available tools
bastionclaw registry list

# Search
bastionclaw registry search calendar

# Install
bastionclaw registry install google-calendar
```

### mcp

Manage MCP (Model Context Protocol) servers.

```bash
bastionclaw mcp <SUBCOMMAND>
```

**Subcommands:**

| Subcommand | Description |
|------------|-------------|
| `list` | List MCP servers |
| `add <URL>` | Add an MCP server |
| `remove <NAME>` | Remove an MCP server |

**Examples:**

```bash
bastionclaw mcp list
bastionclaw mcp add http://localhost:3000/sse
```

### memory

Manage workspace memory.

```bash
bastionclaw memory <SUBCOMMAND>
```

**Subcommands:**

| Subcommand | Description |
|------------|-------------|
| `list` | List documents |
| `read <PATH>` | Read a document |
| `write <PATH>` | Write a document |
| `search <QUERY>` | Search memory |
| `tree` | Show memory tree |

**Examples:**

```bash
# List documents
bastionclaw memory list

# Write a document
echo "My note" | bastionclaw memory write notes/idea.md

# Search
bastionclaw memory search "project idea"
```

### pairing

Manage channel pairing.

```bash
bastionclaw pairing <SUBCOMMAND>
```

**Subcommands:**

| Subcommand | Description |
|------------|-------------|
| `list <CHANNEL>` | List pending requests |
| `approve <CHANNEL> <CODE>` | Approve a pairing request |

**Examples:**

```bash
# List Telegram pending
bastionclaw pairing list telegram

# Approve
bastionclaw pairing approve telegram ABC12345

# List as JSON
bastionclaw pairing list telegram --json
```

### service

Manage system service.

```bash
bastionclaw service <SUBCOMMAND>
```

**Subcommands:**

| Subcommand | Description |
|------------|-------------|
| `install` | Install service |
| `uninstall` | Remove service |
| `start` | Start service |
| `stop` | Stop service |
| `status` | Check service status |

**Examples:**

```bash
# Install
bastionclaw service install

# Check status
bastionclaw service status

# Control
sudo systemctl start bastionclaw  # Linux
brew services start bastionclaw   # macOS
```

### doctor

Run diagnostics.

```bash
bastionclaw doctor [OPTIONS]
```

Checks:
- Database connectivity
- LLM provider access
- Docker availability
- Tunnel detection

**Options:**

| Option | Description |
|--------|-------------|
| `--json` | Output as JSON |
| `--fix` | Attempt to fix issues |

**Example:**

```bash
bastionclaw doctor
bastionclaw doctor --json
```

### status

Show current status.

```bash
bastionclaw status [OPTIONS]
```

Shows:
- Version
- Configuration
- Database status
- LLM backend
- Channels

**Example:**

```bash
bastionclaw status
```

### completion

Generate shell completion scripts.

```bash
bastionclaw completion <SHELL>
```

**Shells:** `bash`, `zsh`, `fish`, `powershell`

**Example:**

```bash
# Bash
bastionclaw completion bash > /etc/bash_completion.d/bastionclaw

# Zsh
bastionclaw completion zsh > /usr/local/share/zsh/site-functions/_bastionclaw
```

## Environment Variables

CLI behavior can be modified with environment variables:

```bash
# Config directory
export BASTIONCLAW_BASE_DIR=/custom/path

# Skip onboarding
export ONBOARD_COMPLETED=true

# Logging
export RUST_LOG=bastionclaw=debug
```

## Exit Codes

| Code | Meaning |
|------|---------|
| `0` | Success |
| `1` | General error |
| `2` | Invalid arguments |
| `3` | Configuration error |
| `4` | Database error |
| `5` | Network error |
