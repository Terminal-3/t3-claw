Run BastionClaw inside a Docker container. This provides isolation and consistency across environments.

<Warning>
**Important distinction:** BastionClaw runs **alongside** Docker (for job sandboxing), not inside Docker by default. This page covers running BastionClaw itself in a container — a different use case from the default installation.
</Warning>

## When to Use Docker

- **Testing**: Quick experiments without modifying your system
- **Consistent environments**: Same configuration across dev/staging/prod
- **Multi-tenant**: Run multiple BastionClaw instances on one host
- **CI/CD**: Automated deployments

## Quick Start

```bash
# Create data directory
mkdir -p ~/.bastionclaw

# Run BastionClaw
docker run -d \
  --name bastionclaw \
  -v ~/.bastionclaw:/home/bastionclaw/.bastionclaw \
  # enabling this mount will allow the container to control the docker daemon, use at own risk
  # -v /var/run/docker.sock:/var/run/docker.sock \
  -p 3000:3000 \
  -p 8080:8080 \
  # Pin to a specific BastionClaw version for reproducible, rollback-friendly deployments.
  nearai/bastionclaw:latest 
```

## Docker Compose

Save as `docker-compose.yml`:

```yaml
version: '3.8'

services:
  bastionclaw:
    # Pin to a specific BastionClaw version for reproducible, rollback-friendly deployments.
    image: nearai/bastionclaw:latest
    container_name: bastionclaw
    restart: unless-stopped

    volumes:
      # Persistent data
      - ~/.bastionclaw:/home/bastionclaw/.bastionclaw

      # Docker socket for sandbox jobs, also exposes your docker host to this container!
      - /var/run/docker.sock:/var/run/docker.sock

    ports:
      # Web Gateway
      - "3000:3000"
      # HTTP Webhook
      - "8080:8080"

    environment:
      # Required: database backend
      - DATABASE_BACKEND=libsql

      # Optional: LLM backend
      - LLM_BACKEND=nearai
      - NEARAI_SESSION_TOKEN=${NEARAI_SESSION_TOKEN}

      # Optional: Web Gateway
      - GATEWAY_ENABLED=true
      - GATEWAY_HOST=0.0.0.0
      - GATEWAY_PORT=3000

      # Optional: HTTP Webhook
      - HTTP_ENABLED=true
      - HTTP_HOST=0.0.0.0
      - HTTP_PORT=8080

# Optional: PostgreSQL instead of libSQL
#  postgres:
#    image: pgvector/pgvector:pg15
#    environment:
#      POSTGRES_USER: bastionclaw
#      POSTGRES_PASSWORD: changeme
#      POSTGRES_DB: bastionclaw
#    volumes:
#      - postgres_data:/var/lib/postgresql/data
#
#volumes:
#  postgres_data:
```

Start:

```bash
docker compose up -d
```

## Volume Mounts

| Host Path | Container Path | Purpose |
|-----------|---------------|---------|
| `~/.bastionclaw` | `/home/bastionclaw/.bastionclaw` | Config, database, logs |
| `/var/run/docker.sock` | `/var/run/docker.sock` | Launch sandbox containers |

## Environment Variables

Pass configuration via environment variables:

```bash
docker run -d \
  --name bastionclaw \
  -e DATABASE_BACKEND=libsql \
  -e LLM_BACKEND=nearai \
  -e NEARAI_SESSION_TOKEN=sess_xxx \
  -e GATEWAY_ENABLED=true \
  -v ~/.bastionclaw:/home/bastionclaw/.bastionclaw \
  -v /var/run/docker.sock:/var/run/docker.sock \
  -p 3000:3000 \
  nearai/bastionclaw:latest
```

See [Configuration Reference](/setup/configuration) for all options.

## Docker-in-Docker Considerations

BastionClaw can launch sandbox containers. In Docker, this requires:

1. **Docker socket mount** (shown above): Allows BastionClaw to launch sibling containers
2. **Privileged mode** (optional): Only if sandbox jobs need elevated permissions

```bash
# With privileged mode (not recommended unless needed)
docker run -d \
  --name bastionclaw \
  --privileged \
  -v ~/.bastionclaw:/home/bastionclaw/.bastionclaw \
  -v /var/run/docker.sock:/var/run/docker.sock \
  ...
```

## Custom Data Directory

To use a different data location:

```bash
mkdir -p /opt/bastionclaw/data

docker run -d \
  --name bastionclaw \
  -e BASTIONCLAW_BASE_DIR=/data \
  -v /opt/bastionclaw/data:/data \
  ...
```

## Reverse Proxy (HTTPS)

For external access, put a reverse proxy in front:

### Caddy

```caddy
# Caddyfile
webg.example.com {
  reverse_proxy localhost:3000
}
```

### nginx

```nginx
server {
  listen 443 ssl;
  server_name webg.example.com;

  ssl_certificate /path/to/cert.pem;
  ssl_certificate_key /path/to/key.pem;

  location / {
    proxy_pass http://localhost:3000;
    proxy_http_version 1.1;
    proxy_set_header Upgrade $http_upgrade;
    proxy_set_header Connection "upgrade";
    proxy_set_header Host $host;
    proxy_set_header X-Real-IP $remote_addr;
  }
}
```

## Updating

```bash
# Pull latest image
docker pull nearai/bastionclaw:latest

# Recreate container
docker stop bastionclaw
docker rm bastionclaw
docker run -d \
  --name bastionclaw \
  -v ~/.bastionclaw:/home/bastionclaw/.bastionclaw \
  -v /var/run/docker.sock:/var/run/docker.sock \
  -p 3000:3000 \
  nearai/bastionclaw:latest

# Or with docker compose
docker compose pull
docker compose up -d
```

## Troubleshooting

<AccordionGroup>
  <Accordion title="Cannot connect to Docker daemon" icon="x-circle">
    Ensure Docker socket is mounted:
    ```bash
    docker run ... -v /var/run/docker.sock:/var/run/docker.sock ...
    ```
  </Accordion>

  <Accordion title="Permission denied on data directory" icon="lock">
    Fix ownership:
    ```bash
    sudo chown -R 1000:1000 ~/.bastionclaw
    ```
  </Accordion>

  <Accordion title="Port already in use" icon="network">
    Change port mapping:
    ```bash
    -p 3002:3000  # Maps host port 3002 to container port 3000
    ```
  </Accordion>
</AccordionGroup>
