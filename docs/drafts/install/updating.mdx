Keep BastionClaw up to date for the latest features and security patches.

## Check Current Version

```bash
bastionclaw --version
# bastionclaw 0.13.0
```

## Update by Installation Method

<Tabs>
  <Tab title="Linux (Shell Script)">
    ```bash
    # Re-run the install script (updates in place)
    curl -fsSL https://install.bastionclaw.ai | bash

    # Verify update
    bastionclaw --version
    ```
  </Tab>

  <Tab title="Ubuntu/Debian (APT)">
    ```bash
    sudo apt update
    sudo apt upgrade bastionclaw
    ```
  </Tab>

  <Tab title="macOS (Homebrew)">
    ```bash
    brew update
    brew upgrade bastionclaw
    ```
  </Tab>

  <Tab title="Docker">
    ```bash
    # Pull latest image
    docker pull nearai/bastionclaw:latest

    # Recreate container
    docker stop bastionclaw
    docker rm bastionclaw
    docker run -d --name bastionclaw ...

    # Or with docker compose
    docker compose pull
    docker compose up -d
    ```
  </Tab>

  <Tab title="From Source">
    ```bash
    cd /path/to/bastionclaw
    git pull origin main
    cargo build --release
    sudo cp target/release/bastionclaw /usr/local/bin/
    ```
  </Tab>
</Tabs>

## Post-Update Steps

### 1. Review Changelog

Check what changed in the new version:

```bash
# View changelog (if installed from source)
cat /usr/local/share/bastionclaw/CHANGELOG.md

# Or online
open https://github.com/nearai/bastionclaw/releases
```

### 2. Run Migrations

Database migrations run automatically on startup, but verify:

```bash
bastionclaw run
# Look for: "Running migrations..." in logs
```

### 3. Restart Service

If running as a service:

```bash
# systemd (Linux)
sudo systemctl restart bastionclaw

# launchd (macOS)
launchctl unload ~/Library/LaunchAgents/ai.bastionclaw.service.plist
launchctl load ~/Library/LaunchAgents/ai.bastionclaw.service.plist

# Homebrew
brew services restart bastionclaw
```

## Downgrading

If you need to rollback:

```bash
# Linux/macOS shell script
# Download specific version
curl -fsSL https://install.bastionclaw.ai | bash -s -- --version 0.12.0

# Docker
docker pull nearai/bastionclaw:v0.12.0
```

<Warning>
Downgrading may require manual database rollback if migrations were applied. Always backup before updating.
</Warning>

## Automatic Updates

### systemd Timer (Linux)

```bash
sudo tee /etc/systemd/system/bastionclaw-update.service <<EOF
[Unit]
Description=BastionClaw Update Check

[Service]
Type=oneshot
ExecStart=/bin/bash -c 'curl -fsSL https://install.bastionclaw.ai | bash'
EOF

sudo tee /etc/systemd/system/bastionclaw-update.timer <<EOF
[Unit]
Description=Run BastionClaw update check daily

[Timer]
OnCalendar=daily
Persistent=true

[Install]
WantedBy=timers.target
EOF

sudo systemctl enable --now bastionclaw-update.timer
```

## Troubleshooting Updates

<AccordionGroup>
  <Accordion title="Update fails with permission denied" icon="lock">
    ```bash
    # Fix permissions
    sudo chown -R $USER:$USER ~/.local/bin/bastionclaw

    # Or update with sudo
    sudo curl -fsSL https://install.bastionclaw.ai | bash
    ```
  </Accordion>

  <Accordion title="Database migration fails" icon="database">
    1. Stop BastionClaw
    2. Backup database
    3. Run with debug logging: `RUST_LOG=debug bastionclaw run`
    4. Check specific migration error
  </Accordion>

  <Accordion title="Service won't start after update" icon="x-circle">
    ```bash
    # Check logs
    sudo journalctl -u bastionclaw -n 50

    # Verify binary
    which bastionclaw
    bastionclaw --version

    # Reinstall if corrupted
    curl -fsSL https://install.bastionclaw.ai | bash
    ```
  </Accordion>
</AccordionGroup>
